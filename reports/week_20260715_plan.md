## 2. 当前方案

当前主链路为：

```text
RGB 图像
  -> 自动食物 mask
  -> 相机内参推断
  -> DepthPro metric depth
  -> 3D scaffold
  -> 支撑平面体积积分
  -> 食物密度 / 热量先验
  -> volume / mass / kcal
```

相比最早的 VoIETA / VoITEX 方案，当前方案不再以 NeRF 重建完整 mesh 作为主路径，而是用单张图深度和支撑平面直接计算体积。这样更适合接入小程序后端，链路更短，推理时间也更可控。

## 4. 模型调研方向

### 4.1 深度估计

- Apple DepthPro：当前已接入，优势是可预测焦距，适合无相机内参场景；问题是模型体量大，约 1.9GB，更适合云端。
- UniDepth / UniDepthV2：目标也是通用 metric depth，并输出相机相关信息，值得作为下一步替代验证对象。
- Metric3D / Metric3D V2：通用 metric depth 方向成熟，泛化能力较强，可作为 DepthPro 的对照实验。
- Depth Anything V2 Metric：模型生态完整，速度和体量选择更多，但 indoor/outdoor 权重对近距离食物场景可能存在尺度偏差，需要实测。

### 4.2 自动分割

- GroundingDINO + SAM / MobileSAM：当前思路是文本提示词定位食物，再生成 mask。
- Grounded-SAM-2：可作为更强的文本驱动分割方案，适合云端。
- SAM 2：分割能力更强，适合图像和视频，但仍需要检测框或提示。
- YOLOE / 开放词表检测分割：如果能直接用文本类别检测食物，可减少 GroundingDINO 依赖。

## 6. 风险

- DepthPro 对近距离桌面食物已有一定效果，但是否能稳定适配中式菜肴仍需实测。
- 无标定物尺度恢复不是完全可靠的物理测量，强依赖深度模型、相机参数和拍摄角度。
- 汤面、酱汁、混合菜不是规则实体，体积和热量都需要类别先验兜底。
- 自动分割如果失败，后续体积计算会直接偏离。
