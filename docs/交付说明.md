# 交付说明

## 1. 本次交付的内容

本文件夹交付的是 `VolETA-MetaFood` 项目中已经整理出来的轻量化“测体积”分支，目标是给小程序后端或师兄的集成代码直接调用。

这一版**不包含**最早期基于 NeRF / COLMAP / Instant-NGP 的重建流程，保留的是当前更适合集成的轻量链路：

- 输入：`RGB 图像`
- 可选补充：`food class / 手机型号 / 已有 mask / 已有相机内参`
- 中间过程：`metric depth -> 3D scaffold -> 支撑平面定尺 / 先验补偿`
- 输出：`体积估计 + 热量估计 + 可视化结果 + JSON 摘要`

## 3. 目录结构

### 核心脚本

- `scripts/run_metric_depth_demo.py`
  - 调用本地 metric depth 模型，对单张图预测深度并输出深度图、可视化和 scaffold 摘要。

- `scripts/run_depthpro_demo.py`
  - 调用 Apple DepthPro 做绝对尺度深度预测。
  - 如果没有手动提供 intrinsics，会尝试通过手机型号配置、图片 EXIF 或 DepthPro 预测焦距自动回填。

- `scripts/infer_intrinsics_from_image.py`
  - 从手机型号配置、图片 EXIF 或焦距兜底值推断 intrinsics。

- `scripts/run_grounded_sam_to_voleta.py`
  - 使用文本提示词自动生成食物 mask。

- `scripts/run_auto_food_volume_demo.py`
  - 一键入口：自动 mask -> 自动 intrinsics -> DepthPro -> 体积估计。

- `scripts/build_noref_scaffold.py`
  - 根据深度结果构建三维点支架（scaffold），供后续体积计算使用。

- `scripts/estimate_metric_volume_calorie.py`
  - 使用 metric depth + 支撑平面方式估计食物体积和热量。

- `scripts/estimate_noref_scale_calorie.py`
  - 当 metric depth 不稳定时，使用无参照先验做兜底估计。

### 配置与先验

- `assets/noref_food_priors.json`
  - 存放食物类别对应的尺寸、密度、热量先验。

- `assets/phone_camera_profiles.example.json`
  - 手机型号与相机参数的示例映射表。

### 示例输入

- `examples/input/example_image.jpg`
- `examples/input/example_mask.jpg`
- `examples/input/intrinsics.json`

### 示例输出

- `examples/output_support_plane/`
  - 支撑平面体积估计的示例结果

## 4. 这次交付里不包含什么

为了减小体积、降低耦合，这次没有放入以下内容：

- 老版本 NeRF / COLMAP / Instant-NGP 重建代码
- 自动分割的大型 vendor 仓库
- 当前工作区中不存在的模型权重文件

也就是说，这次交付的重点是“测体积模块”，不是整套“识别 + 分割 + 体积 + 热量”的完整产品代码。

## 5. 运行前需要准备什么

### Python 依赖

先安装：

```powershell
pip install -r requirements.txt
```

### 模型权重

当前脚本需要你本地自行准备对应的深度模型：

- `run_metric_depth_demo.py`
  - 需要一个本地 Hugging Face 风格的 metric depth 模型目录

- `run_depthpro_demo.py`
  - 需要 `depth_pro.pt`

这些权重本次没有一并打包进去。

### 自动分割依赖

如果要使用自动 mask 功能，还需要本地准备 GroundingDINO / MobileSAM 对应依赖和权重。
这部分为了减小交付体积，没有随本包一并提供。

## 6. 标准输入输出约定

### 输入

后端集成时，至少需要准备三类输入：

1. 一张 RGB 食物图像
2. 可选二值 mask（如果没有，可走自动分割）
3. 可选相机内参 JSON（如果没有，可走自动推断）
4. 可选手机型号

### 输出

脚本会输出：

- 深度图或三维支架结果
- 支撑平面可视化图
- 高度图
- 体积 / 热量估计 JSON

适合后端直接读取 JSON，再把图像结果作为调试材料或可视化附件。

## 7. 新增的一键自动输入方式

现在这套交付包已经支持更接近手机端的输入方式。

最简使用时，理论上只需要：

1. 一张食物图片
2. 可选食物类别
3. 可选手机型号

即可通过 `run_auto_food_volume_demo.py` 自动完成：

- 生成食物 mask
- 推断 intrinsics
- 运行 DepthPro
- 输出体积 / 热量结果

## 8. 当前方案的边界

这次交付的是一个可运行、可集成的“体积估计核心模块”，但仍有几个边界需要说明：

- 它默认上游已经给出 mask，本身不负责完整的食物检测与分割
- 自动 mask 依赖文本提示词和现成分割模型，在复杂中餐、混合拼盘、汤汁遮挡场景下仍可能不稳定
- 绝对尺度恢复效果很依赖深度模型权重是否适合“近距离桌面食物”场景
- 支撑平面方案对“放在盘子里、桌面上”的食物更稳定
- 对汤面、混合菜、强遮挡、强反光场景仍需要进一步优化
